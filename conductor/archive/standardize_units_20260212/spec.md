# Specification: Standardize Tag Size to Meters (Float)

## Overview
To prevent "Mars Climate Orbiter" style unit mismatch risks and ensure frictionless interop with `locus-tag` (Rust) and Blender, this track standardizes the representation of tag sizes to Meters (Float) using the field name `tag_size_m`.

## Functional Requirements
- **Schema Update**: Update `DatasetManifest` in `src/render_tag/orchestration/experiment_schema.py` to use `tag_size_m: float` instead of `tag_size_mm: int`.
- **Config Mapping**: Ensure the `dataset_info.json` generation logic correctly pulls the tag size from `TagConfig.size_meters` and maps it to the `tag_size_m` field in the manifest.
- **Metadata Consistency**: Update `src/render_tag/audit/dataset_info.py` to ensure all exported metadata adheres to the `tag_size_m` naming convention.
- **Manifest Refresh**: Update `configs/experiments/locus_pose_baseline.yaml` to use the standardized `tag_size_m` field.

## Non-Functional Requirements
- **Numerical Precision**: Use floating-point values to match Blender and `locus-tag` physics engines.
- **System Safety**: Minimize the risk of 1000x scaling errors by removing the "mm" unit from core schemas.

## Acceptance Criteria
- `DatasetManifest` successfully validates configurations using `tag_size_m` (float).
- `dataset_info.json` generated by `render-tag experiment run` contains the `tag_size_m` field.
- The Phase 2 baseline experiments (`locus_pose_baseline.yaml`) run without error using the updated schema.

## Out of Scope
- Backwards compatibility for `tag_size_mm` (this is a breaking change to ensure safety).

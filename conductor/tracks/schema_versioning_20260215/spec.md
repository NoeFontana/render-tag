# Specification: Strict Schema Versioning

## Overview
To ensure long-term reproducibility and facilitate safe refactoring of internal data structures, this track introduces strict schema versioning for `GenConfig` (YAML inputs) and `JobSpec` (JSON manifests). A centralized `SchemaMigrator` will detect, patch, and upgrade legacy configurations to the current standard during the Resolution Phase.

## Functional Requirements
- **Mandatory Version Field:** Add a top-level `version` string field to both `GenConfig` and `JobSpec` Pydantic models.
- **Legacy Detection:** Treat any configuration file missing the `version` field as version `"0.0"`.
- **Sequential Migration:** Implement a `SchemaMigrator` that applies version-specific transformation functions (e.g., `0.0 -> 1.0`) to raw dictionaries before Pydantic validation.
- **On-Disk Upgrades:** If a file is successfully migrated, the updated dictionary (including the new version field) must be written back to the original file path on disk.
- **Pipeline Integration:** Integrate the migration logic into the `load_config` utility and the `JobSpec` loading sequence used by the CLI and orchestrator.

## Non-Functional Requirements
- **Reproducibility:** Migration must be deterministic; the same legacy file must always result in the same versioned output.
- **Observability:** The system must log an `INFO` or `WARNING` level message when a file is being migrated, specifying the file path and the version change.
- **Robustness:** Failure during migration (e.g., impossible transformation) should halt the pipeline with a clear error message rather than proceeding with corrupted data.

## Acceptance Criteria
- [ ] Running `render-tag generate` with an unversioned YAML config succeeds.
- [ ] After execution, the YAML config file on disk contains `version: "1.0"`.
- [ ] Attempting to load an old `job_spec.json` (from before this change) via `render-tag audit` or `render-tag viz` succeeds after automatic migration.
- [ ] New `job_spec.json` files generated by the pipeline include the mandatory `version` field.

## Out of Scope
- Automatic migration of rendered dataset sidecar files (metadata JSONs per image).
- Support for downgrading schemas.
- Complex SemVer logic (simple string matching or float-like comparison is sufficient for the initial implementation).
